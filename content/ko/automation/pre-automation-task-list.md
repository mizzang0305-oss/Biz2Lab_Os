## 도구를 먼저 고르면 빠지는 정보

자동화 상담에서 “메일을 시트로 옮기고 싶다”는 요청을 자주 볼 수 있습니다. 하지만 실제 업무를 보면 메일마다 양식이 다르고, 첨부파일을 열어야 하며, 재고가 없을 때 담당자에게 묻는 과정이 있습니다. 메일을 옮기는 기능만 만들면 핵심 예외는 그대로 사람에게 남습니다.

업무 목록은 기능 목록이 아닙니다. **일이 시작되는 순간부터 완료를 확인하는 순간까지 실제로 무엇이 일어나는지** 기록하는 문서입니다.

## 5일 동안 관찰할 항목

다음 항목을 업무가 발생할 때마다 적습니다.

| 관찰 항목 | 기록 예시 |
| --- | --- |
| 시작 조건 | 고객 전화가 오면 시작 |
| 입력 원본 | 통화 메모, 상품코드, 연락처 |
| 처리 순서 | 메모 → 재고 확인 → 주문표 입력 |
| 기다림 | 창고 답변 12분 |
| 예외 | 재고 부족, 주소 불명확 |
| 완료 조건 | 주문번호와 약속시간 확정 |
| 사용 도구 | 전화, 메신저, 시트 |
| 재작업 | 상품코드 오입력으로 6분 추가 |

첫날에는 자세하게 적고, 둘째 날부터 반복되는 단계에 표시합니다. 금요일에는 업무별 발생 횟수와 총시간을 합칩니다. 기다림 시간은 자동화로 줄일 수 있는 처리시간과 분리해야 합니다.

## 업무 이름을 행동으로 적기

`고객 관리`, `매출 관리`, `문서 업무`처럼 큰 이름은 자동화 범위를 정하기 어렵습니다. 다음처럼 시작과 결과가 보이게 바꿉니다.

- 고객 관리 → 상담 후 다음 연락일 기록
- 매출 관리 → 전일 입금과 주문 금액 대조
- 문서 업무 → 회의 원문에서 결정사항 표 작성
- 주문 처리 → 전화 주문을 통합 주문번호로 등록

행동 단위로 적으면 한 업무 안에서도 자동화 가능한 부분과 사람이 판단할 부분이 나뉩니다.

## 예외를 숨기지 않는 인터뷰

담당자에게 “평소에는 어떻게 하나요?”라고 묻는 것만으로 부족합니다. 다음 질문이 실제 예외를 찾는 데 도움이 됩니다.

- 지난주에 가장 오래 걸린 건은 무엇이 달랐나요?
- 원본 정보가 없을 때 누구에게 확인하나요?
- 잘못 처리한 사실은 어디에서 발견하나요?
- 고객이 급하다고 하면 순서가 어떻게 바뀌나요?
- 담당자가 없으면 다른 사람이 이어서 할 수 있나요?
- 완료됐다고 표시한 뒤 다시 열어 보는 경우는 언제인가요?

예외를 자동화에서 빼는 것이 아니라, 예외가 발생하면 수동 대기열로 보내는 규칙을 만들기 위해 묻는 질문입니다.

## 목록을 네 가지 결과로 분류하기

5일 기록을 마치면 각 업무를 다음 중 하나로 분류합니다.

1. **즉시 정리**: 양식과 완료 기준만 통일해도 좋아지는 업무
2. **규칙 자동화 후보**: 입력과 결과가 일정하고 예외가 적은 업무
3. **AI 초안 후보**: 요약·분류처럼 사람이 최종 판단해야 하는 업무
4. **보류**: 발생이 드물거나 위험이 높아 자동화 비용이 더 큰 업무

보류가 실패를 뜻하지 않습니다. 자동화하지 않을 일을 명확하게 정하면 중요한 후보에 시간을 집중할 수 있습니다.

## 완료 기준을 테스트하는 방법

다른 담당자에게 업무 목록만 보여주고 같은 결과를 만들 수 있는지 확인합니다. 질문이 많이 나오면 입력자료나 완료 조건이 부족한 것입니다. 예를 들어 “주문 등록 완료”라고만 쓰면 재고 확인과 납기 약속이 빠질 수 있습니다.

자동화는 불명확한 업무를 명확하게 만들어 주지 않습니다. 불명확한 단계를 더 빠르게 반복할 뿐입니다. 5일 관찰 기록이 도구 선택보다 먼저 필요한 이유입니다.
