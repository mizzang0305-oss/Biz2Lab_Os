## 문서 정리의 목표는 짧게 만드는 것이 아니다

회의 메모를 한 페이지로 줄였는데 결정사항이 빠졌다면 정리에 실패한 것입니다. 상담 기록이 깔끔해졌지만 다음 연락일이 없다면 업무로 이어지지 않습니다. ChatGPT 문서 정리의 목표는 문장을 매끄럽게 만드는 것이 아니라 **확정된 사실, 확인이 필요한 내용, 다음 행동을 분리하는 것**입니다.

AI가 만든 문서는 원문을 대체하지 않습니다. 원문은 증거와 맥락을 보존하고, 정리본은 일을 빠르게 이어가기 위한 색인 역할을 합니다.

## 원문·초안·확정본을 세 파일로 나누기

문서 이름부터 구분하면 실수를 줄일 수 있습니다.

- `회의명_원문_2026-07-16.txt`: 수정하지 않은 기록
- `회의명_AI초안_2026-07-16.md`: AI가 구조화한 내용
- `회의명_확정_2026-07-16.md`: 참석자가 확인한 최종본

AI 초안을 확정본 위에 덮어쓰지 않습니다. 나중에 숫자나 담당자가 틀렸을 때 원문과 비교할 수 있어야 합니다. 공유 폴더에서는 파일명뿐 아니라 `검토중`, `확정`, `폐기` 상태도 표시합니다.

## 회의 메모를 정리하는 실제 요청 형식

“요약해 줘”라고만 요청하면 중요도를 AI가 임의로 판단합니다. 다음처럼 결과 칸을 먼저 정합니다.

> 아래 원문에서 결정사항, 담당자, 기한, 미확정 질문을 분리하라. 원문에 없는 내용은 추측하지 말고 `확인 필요`로 표시하라. 금액과 날짜는 원문 표현을 그대로 인용하고, 마지막에 누락 가능성이 있는 항목을 적어라.

결과를 받은 뒤에는 다음 순서로 검토합니다.

1. 결정사항이 실제 발언과 일치하는지 봅니다.
2. 담당자가 확정되지 않은 일을 임의로 배정하지 않았는지 확인합니다.
3. 날짜와 금액을 원문에서 다시 찾습니다.
4. 질문과 결정이 섞이지 않았는지 봅니다.
5. 확정본에 검토자와 검토일을 기록합니다.

## 상담 메모는 고객 약속을 중심으로 정리하기

상담 기록에는 분위기나 장황한 대화보다 약속 내용이 중요합니다. 다음 항목이면 충분합니다.

| 항목 | 예시 |
| --- | --- |
| 고객이 요청한 것 | A상품 10개, 금요일 이전 납품 가능 여부 |
| 우리가 확인할 것 | 현재 재고와 택배 마감시간 |
| 약속한 다음 행동 | 오늘 16시까지 가능 여부 회신 |
| 확정되지 않은 것 | 할인율과 최종 배송비 |
| 원문 위치 | 상담 녹취 08:20~11:40 |

AI가 할인율이나 납기를 자연스럽게 만들어 내지 않도록 `확정되지 않은 것`을 별도 항목으로 둡니다. 고객에게 보낼 답장은 확정된 정보만 사용합니다.

## 개인정보와 비공개 자료를 넣기 전 확인

개인용 ChatGPT와 조직용 계정은 데이터 관리 조건이 다를 수 있습니다. 사용 전에 현재 계정의 데이터 설정과 회사 정책을 확인해야 합니다. OpenAI는 ChatGPT 설정에서 대화가 모델 개선에 사용되는지 관리하는 데이터 제어 기능을 안내합니다. [OpenAI 데이터 제어 안내](https://help.openai.com/en/articles/7730893-chat-and-file-retention-policies-in-chatgpt)

고객 실명, 연락처, 계좌정보, 계약 원문, 인증정보는 필요하지 않다면 입력하지 않습니다. 예시가 필요하면 `고객A`, `샘플계약`, `100만 원`처럼 가상 값으로 바꿉니다. 삭제했다고 끝나는 것이 아니라 어떤 정보를 제거했는지 작업 기록에 남기는 편이 좋습니다.

## 잘 정리된 문서의 판정 기준

문장이 자연스러운지는 마지막 기준입니다. 먼저 다음을 확인합니다.

- 원문으로 돌아갈 수 있다.
- 원문에 없는 사실이 확정 문장처럼 들어가지 않았다.
- 누가 무엇을 언제까지 하는지 보인다.
- 확인이 필요한 항목이 숨겨지지 않았다.
- 비공개 정보가 불필요하게 복사되지 않았다.

이 기준을 통과한 뒤에 문장 길이와 표현을 다듬습니다. AI가 잘 쓰는 문장을 만드는 것보다 사람이 안전하게 확인할 수 있는 문서를 만드는 것이 우선입니다.
