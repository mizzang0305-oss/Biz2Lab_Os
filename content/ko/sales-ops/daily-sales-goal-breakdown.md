## 월 목표를 오늘 숫자로 바꾸는 세 식

매출 목표가 있어도 오늘 무엇을 해야 하는지 알 수 없다면 운영 숫자로 기능하지 못합니다. 다음 세 값을 매일 같은 시간에 계산합니다.

`부족금액 = 월 목표 - 현재 인정 실적`

`하루 필요매출 = 부족금액 ÷ 남은 영업일`

`필요 주문수 = 하루 필요매출 ÷ 평균 객단가`

현재 인정 실적은 주문, 출고, 세금계산서, 입금 중 하나로 기준을 정합니다. 팀마다 서로 다른 기준을 쓰면 같은 목표를 보고도 다른 숫자가 나옵니다.

## 1,000만 원 목표 예시

월 목표가 1,000만 원이고 현재 인정 실적이 650만 원이라면 부족금액은 350만 원입니다. 남은 영업일이 7일이면 하루에 평균 50만 원이 필요합니다. 평균 객단가가 5만 원이면 하루 필요 주문수는 10건입니다.

이 계산은 미래를 보장하는 예측이 아닙니다. 지금 속도와 남은 시간을 비교하기 위한 기준선입니다. 하루 매출이 50만 원에 못 미쳤다면 다음 날 목표를 다시 계산합니다.

| 항목 | 값 |
| --- | ---: |
| 월 목표 | 10,000,000원 |
| 현재 인정 실적 | 6,500,000원 |
| 부족금액 | 3,500,000원 |
| 남은 영업일 | 7일 |
| 하루 필요매출 | 500,000원 |
| 평균 객단가 | 50,000원 |
| 필요 주문수 | 10건 |

## 주문수로 끝내지 말고 행동량으로 한 번 더 나누기

필요 주문수가 10건이라고 해서 “오늘 10건 판매”만 적으면 실행 방법이 없습니다. 과거 전환 흐름을 기준으로 문의, 견적, 재구매 연락 같은 행동으로 나눕니다.

예를 들어 문의 4건 중 주문 1건이 발생했다면 신규 주문 6건을 만들기 위해 약 24건의 유효 문의가 필요합니다. 기존 고객 재주문 4건을 예상한다면 신규 문의 목표는 줄어듭니다. 이 비율은 업종마다 다르므로 자신의 기록으로 계산해야 합니다.

행동 계획의 예:

- 미결 견적 3건 확인
- 입금 대기 주문 2건 일정 확인
- 재주문 가능 고객 8명에게 필요한 정보 제공
- 예약 취소분을 다시 판매할 수 있는지 확인
- 재고 부족으로 놓치는 상품이 있는지 점검

## 영업일을 잘못 세면 목표가 왜곡된다

남은 달력 일수가 아니라 실제 영업 가능한 날을 사용합니다. 휴무일, 배송 불가일, 대형 행사 준비일은 일반 영업일과 다를 수 있습니다. B2B 영업이라면 거래처의 결재일과 납품 가능일도 고려해야 합니다.

남은 영업일이 0인데 부족금액이 남았다면 0으로 나눌 수 없습니다. 이때는 목표 미달을 숨기지 말고 다음 기간 이월 여부와 원인을 기록합니다.

## 좋은 날과 나쁜 날을 평균으로 숨기지 않기

주말 매출이 평일의 두 배인 매장이라면 하루 목표를 동일하게 나누면 평일마다 실패처럼 보입니다. 요일별 기준을 별도로 만들거나, 주간 단위로 배분한 뒤 하루 계획을 세우는 편이 낫습니다.

대형 주문 한 건이 전체 실적을 끌어올렸다면 반복 가능한 실적인지 표시합니다. 일회성 매출을 포함한 평균 객단가로 필요 주문수를 계산하면 다음 기간 계획이 지나치게 낮아질 수 있습니다.

## 매일 10분 점검 순서

1. 전일 취소와 반품을 반영해 인정 실적을 확정합니다.
2. 남은 영업일을 확인합니다.
3. 부족금액과 하루 필요매출을 다시 계산합니다.
4. 평균 객단가 기준 필요 주문수를 봅니다.
5. 오늘 확정된 주문과 아직 필요한 주문을 분리합니다.
6. 숫자를 바꿀 수 있는 다음 행동을 한 문장으로 적습니다.

목표는 압박용 숫자가 아니라 행동을 선택하는 도구입니다. 계산만 반복하고 다음 행동이 바뀌지 않는다면 보고 방식부터 다시 설계해야 합니다.
