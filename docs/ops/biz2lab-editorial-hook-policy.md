# Biz2Lab Editorial Hook Policy

## 핵심 원칙

Biz2Lab 콘텐츠의 첫 화면은 독자가 "이걸 안 보면 손해다"라고 느낄 만큼 무시 비용을 분명하게 보여줘야 합니다.

이 원칙은 클릭베이트가 아닙니다. 과장, 공포, 허위 성과 약속, 보장 표현을 금지하고 실제 업무 손실, 시간 낭비, 비용 증가, 운영 리스크, 기회비용, 도입 실패 가능성을 구체적으로 설명합니다.

## 적용 위치

- 제목: 주제와 함께 놓치면 생기는 비용 또는 판단 기준을 보여줍니다.
- meta description: 검색자가 이 글을 읽지 않으면 어떤 시간, 비용, 리스크를 놓칠 수 있는지 답합니다.
- 첫 문단: 독자가 지금 겪는 문제 또는 놓치고 있는 손실을 먼저 말합니다.
- H2/H3: 기능 나열보다 도입 전 확인할 기준, 줄일 수 있는 시간, 놓치면 손해 보는 한계를 드러냅니다.
- 내부 링크: 단순한 관련 글보다 다음 판단에 필요한 글이라는 맥락을 줍니다.
- CTA: 바로 구매나 도입을 압박하지 않고, 먼저 줄일 수 있는 시간·비용·운영 리스크를 점검하게 합니다.

## 금지 사항

- 무조건 추천, 완전 무료, 상업 사용 보장 같은 확정 표현
- 공포를 조장하는 과장 문구
- 실제 데이터 없이 Search Console, GA4, CTR, 순위, 조회수 수치 제시
- meta keywords 추가
- 법률, 라이선스, self-hosting 주의 섹션 제거
- slug, route, canonical 변경
